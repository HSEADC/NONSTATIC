Кинетическая типографика

Кинетическая типографика — это искусство и техника анимации текста. В среде программирования Processing она становится доступной и позволяет создавать уникальные визуальные эффекты, используя текстовые анимации. Такие анимации можно использовать в различных проектах, например, в постерах.
Настя Абазина
16/02/20025

Алгоритм

Шаг 1
Сначала объявляем переменные. Тебе нужно использовать PGraphics для рендеринга текста, PFont  для шрифта, PImage для фона и waveOffset для управления анимацией. Количество тайлов (плиток) задается через tilesX и tilesY — они определяют, на сколько частей делится изображение.
Скопируй
PGraphics pg;
PFont font;
float waveOffset;
int tilesX = 16;
int tilesY = 24;
PImage bg;
Шаг 2
В setup() происходит начальная настройка анимации. Строка size(1115, 700, P2D) создает окно размером 1115
Размер холста ты можешь задать исходя из размеров картинки
Затем если ты хочешь загрузить фоновое изображение, то можно вставить ссылку. Если интернета нет или ссылка битая, в блоке catch создается черный фон. Картинка автоматически растягивается под размер окна с помощью bg.resize(width, height).
Скопируй
void setup() {
  size(1115, 700, P2D);
  font = createFont("Helvetica", 200);
  pg = createGraphics(width, height, P2D);

  try {
    bg = loadImage("https://
  i.pinimg.com/736x/31/4f/32/314f3220e6c8bbb205e.jpg");
    bg.resize(width, height); 
  } catch (Exception e) {
    println("Ошибка загрузки изображения: " + e.getMessage());    
    bg = createImage(width, height, RGB);
  }
}
Шаг 3
Функция draw() выполняется 60 раз в секунду. Каждый раз мы начинаем с отрисовки фона: image(bg, 0, 0). Затем увеличиваем waveOffset на 0.04 — это «двигатель» анимации.
Если увеличить это число, волны побегут быстрее
Скопируй
void setup() {
  void draw() {
    image(bg, 0, 0);
    waveOffset += 0.04;
  }
}
Шаг 4
Раз в секунду (когда frameCount делится на 60 без остатка) мы обновляем текст в буфере pg. Начини с pg.beginDraw(), чтобы подготовить буфер к работе. pg.clear() стирает старый текст, а pg.fill(#ECFF0C) задает твой выбранный цвет. Центрируй текст с помощью pg.textAlign(CENTER, CENTER). Надпись «JUST\nTIRED» располагается с переносом строки, а pg.endDraw() завершает работу с буфером. 
Скопируй
if (frameCount % 60 == 0) {
pg.beginDraw();
pg.clear();
pg.fill(#ECFF0C);
pg.textFont(font);
pg.textAlign(CENTER, CENTER);
pg.textSize(180);
pg.pushMatrix();
pg.translate(width/2, height/2 - 80);
pg.text("JUST\nTIRED", 0, 0);
pg.popMatrix();
pg.endDraw();
}
Шаг 5
Разбей текст на плитки размером tileW и tileH (ширина и высота одной плитки). Для каждой плитки вычисли смещение по вертикали и горизонтали с помощью функций sin() и cos(). Например, sin(waveOffset + y * 0.2) * 30 создает вертикальное смещение: чем ниже плитка (больше y), тем больше задержка, что дает эффект бегущей волны. Горизонтальное смещение cos(waveOffset * 1.5 + x * 0.3) * 40 работает похоже, но волна здесь движется быстрее за счет множителя 1.5. 
Скопируй
int tileW = width/tilesX;
int tileH = height/tilesY;

for (int y = 0; y < tilesY; y++) {
    float yOffset = sin(waveOffset + y*0.2) * 30;
    for (int x = 0; x < tilesX; x++) {
      float xOffset = cos(waveOffset*1.5 + x*0.3) * 40;
Шаг 6
Оператор % pg.width обеспечивает циклическое смещение плиток: если элемент выходит за границу экрана, он появляется с противоположной стороны. Функция tint(255, 220) задает прозрачность плиток, позволяя фону просвечивать. copy() переносит фрагменты из буфера на экран, а blendMode(NORMAL) сбрасывает параметры смешивания цветов для нового кадра. 
Скопируй
float sx = (x * tileW + xOffset + pg.width) % pg.width;
float sy = (y * tileH + yOffset + pg.height) % pg.height;
      
tint(255, 220);
      copy(pg, (int)sx, (int)sy, tileW, tileH, x*tileW, y*tileH, tileW, tileH);
    }
  }
  
  blendMode(NORMAL);
}

Кастомизация

Ты можешь поэксперементировать, например поменяй цвет текста, заменив #ECFF0C на свой. Ускорь волну, увеличив значение 0.04 в строке waveOffset += 0.04 до 0.07. Сделай плитки крупнее, уменьшив tilesX до 12 и tilesY до 18.
